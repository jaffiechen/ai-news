## 快速开始

添加新数据源只需 3 步：

1. **创建 Fetcher 文件**：在 `scripts/fetcher/fetchers/` 目录下创建新的 fetcher 类
2. **注册 Fetcher**：在 `scripts/fetcher/fetchers/index.ts` 中导出并加入 `createAllFetchers()`
3. **前端配置**：在 `src/components/settings/SourceModal.vue` 的 `SITE_INFO` 中添加站点描述

## 核心概念

### Fetcher 基类

所有数据源都继承自 `BaseFetcher` 抽象类，位于 `scripts/fetcher/fetchers/base.ts`。

**必须实现的属性：**
- `siteId: string` - 站点唯一标识（小写，用连字符分隔，如 `my-site`）
- `siteName: string` - 站点显示名称

**必须实现的方法：**
- `fetch(now: Date): Promise<RawItem[]>` - 抓取数据的核心方法

### RawItem 数据结构

每条新闻必须包含以下字段：

```typescript
interface RawItem {
  siteId: string;      // 站点 ID（与 Fetcher 的 siteId 一致）
  siteName: string;    // 站点名称（与 Fetcher 的 siteName 一致）
  source: string;      // 来源名称（子分类，如 "Hacker News · 24h最热"）
  title: string;       // 新闻标题
  url: string;         // 新闻链接
  publishedAt: Date | null;  // 发布时间
  meta?: Record<string, unknown>;  // 额外元数据（可选）
}
```

使用 `this.createItem()` 辅助方法创建 RawItem：

```typescript
this.createItem({
  source: '来源名称',
  title: '新闻标题',
  url: 'https://example.com/article',
  publishedAt: new Date(),
  meta: { /* 可选 */ }
})
```

## 添加步骤

### 第 1 步：创建 Fetcher 文件

在 `scripts/fetcher/fetchers/` 目录下创建新文件，命名规则为 `站点id.ts`（如 `mysite.ts`）。

参考 [Fetcher 模板](#fetcher-模板) 编写代码。

### 第 2 步：注册 Fetcher

编辑 `scripts/fetcher/fetchers/index.ts`：

```typescript

export { MySiteFetcher } from './mysite.js';

// 2. 导入
import { MySiteFetcher } from './mysite.js';

// 3. 在 createAllFetchers() 数组中添加实例
export function createAllFetchers(): Fetcher[] {
  return [
    // ... 其他 fetcher
    new MySiteFetcher(),  // 添加这一行
  ];
}
```

### 第 3 步：配置前端显示

编辑 `src/components/settings/SourceModal.vue`，在 `SITE_INFO` 对象中添加：

```typescript
const SITE_INFO: Record<string, { description: string; url: string }> = {
  // ... 其他站点
  'my-site': {
    description: '站点描述，一句话介绍这个数据源',
    url: 'https://example.com/',  // 数据源官网地址，没有则留空字符串
  },

```

## Fetcher 模板

### 最简模板（JSON API）

```typescript
import type { RawItem } from '../types.js';
import { BaseFetcher } from './base.js';

export class MySiteFetcher extends BaseFetcher {
  siteId = 'my-site';
  siteName = '我的站点';

  async fetch(now: Date): Promise<RawItem[]> {
    const data = await this.fetchJsonData<any>(
      'https://api.example.com/news'
    );
    
    const items: RawItem[] = [];
    for (const item of data.items || []) {
      const title = (item.title || '').trim();
      const url = (item.url || '').trim();
      if (!title || !url) continue;

      const publishedAt = this.parseDate(item.published_at) || now;
      
      items.push(
        this.createItem({
          source: item.source || this.siteName,
          title,
          url,
          publishedAt,
        })
      );
    }

    return items;
  }
}
```

## 常见类型示例

### 1. RSS 订阅源

```typescript
import type { RawItem } from '../types.js';
import { BaseFetcher } from './base.js';
import { fetchText } from '../utils/http.js';
import Parser from 'rss-parser';

const parser = new Parser();

export class MyRssFetcher extends BaseFetcher {
  siteId = 'my-rss';
  siteName = '我的RSS';

  async fetch(now: Date): Promise<RawItem[]> {
    const xml = await fetchText('https://example.com/feed.xml');
    const feed = await parser.parseString(xml);
    
    const items: RawItem[] = [];
    for (const item of feed.items) {
      const title = (item.title || '').trim();
      const url = (item.link || '').trim();
      if (!title || !url) continue;

      const publishedAt = item.pubDate ? new Date(item.pubDate) : now;
      
      items.push(
        this.createItem({
          source: item.creator || this.siteName,
          title,
          url,
          publishedAt,
        })
      );
    }

    return items;
  }
}
```

### 2. HTML 页面抓取

```typescript
import type { RawItem } from '../types.js';
import { BaseFetcher } from './base.js';
import * as cheerio from 'cheerio';

export class MyHtmlFetcher extends BaseFetcher {
  siteId = 'my-html';
  siteName = '我的HTML站点';

  async fetch(now: Date): Promise<RawItem[]> {
    const $ = await this.fetchHtml('https://example.com/news');
    
    const items: RawItem[] = [];
    $('.news-item').each((_, el) => {
      const $el = $(el);
      const title = $el.find('.title').text().trim();
      const url = $el.find('a').attr('href') || '';
      if (!title || !url) return;

      const dateStr = $el.find('.date').text().trim();
      const publishedAt = this.parseDate(dateStr) || now;
      
      items.push(
        this.createItem({
          source: this.siteName,
          title,
          url: url.startsWith('http') ? url : `https://example.com${url}`,
          publishedAt,
        })
      );
    });

    return items;
  }
}
```

### 3. 多频道/多分类

```typescript
import type { RawItem } from '../types.js';
import { BaseFetcher } from './base.js';
import pLimit from 'p-limit';

const CATEGORIES = [
  { id: 'tech', name: '科技' },
  { id: 'ai', name: 'AI' },
];

export class MultiCategoryFetcher extends BaseFetcher {
  siteId = 'multi-cat';
  siteName = '多分类站点';

  async fetch(now: Date): Promise<RawItem[]> {
    const limit = pLimit(3); // 并发限制
    const items: RawItem[] = [];

    const results = await Promise.all(
      CATEGORIES.map((cat) =>
        limit(async () => {
          try {
            const data = await this.fetchJsonData<any>(
              `https://api.example.com/news?category=${cat.id}`
            );
            return { category: cat, items: data.items || [] };
          } catch {
            return { category: cat, items: [] };
          }
        })
      )
    );

    for (const { category, items: catItems } of results) {
      for (const item of catItems) {
        const title = (item.title || '').trim();
        const url = (item.url || '').trim();
        if (!title || !url) continue;

        items.push(
          this.createItem({
            source: category.name,
            title,
            url,
            publishedAt: this.parseDate(item.date) || now,
          })
        );
      }
    }

    return items;
  }
}
```

## 验证与测试

### 运行抓取测试

```bash
npx tsx scripts/fetcher/index.ts
```

在输出中检查你的数据源：
- ✅ `[你的站点名] XX items` - 抓取成功
- ❌ `[你的站点名] Failed: ...` - 抓取失败，检查错误信息

### 检查输出文件

抓取成功后，数据会写入 `public/data/` 目录：
- `latest-24h.json` - 24 小时内的数据
- `latest-7d.json` - 7 天内的数据
- `source-status.json` - 数据源状态

### 启动前端验证

```bash
npm run dev
```

访问 `http://localhost:5173/`，检查：
1. 数据源统计卡片中平台数量是否增加
2. 点击"详情 →"查看数据源概览，确认新站点已列出
3. 时间轴中是否有来自新数据源的新闻

## 前端显示配置

### 数据源概览弹窗

在 `src/components/settings/SourceModal.vue` 中配置：

```typescript
const SITE_INFO: Record<string, { description: string; url: string }> = {
  'your-site-id': {
    description: '一句话描述这个数据源是什么',
    url: 'https://example.com/',  // 官网地址
  },
}
```

### 注意事项

1. **siteId 必须一致**：Fetcher 中的 `siteId` 与 `SITE_INFO` 中的 key 必须完全一致
2. **命名规范**：siteId 使用小写字母和连字符，如 `my-awesome-site`
3. **错误处理**：Fetcher 应妥善处理异常，单个数据源失败不应影响整体抓取
4. **并发控制**：多个请求时使用 `p-limit` 限制并发数，避免被封 IP
5. **超时设置**：HTTP 请求建议设置合理的超时时间（15-30 秒）

## 故障排查

### 问题：抓取返回 0 条

**可能原因：**
- URL 不正确
- 返回的数据结构与预期不符
- 选择器写错了（HTML 抓取）

**调试方法：**
```typescript
// 打印返回数据看看结构
console.log('data:', JSON.stringify(data, null, 2));
```

### 问题：CORS 或网络错误

**解决方法：**
- 检查 URL 是否可访问
- 添加 User-Agent header
- 使用代理（如需要）

### 问题：时间解析错误

**解决方法：**
- 使用 `this.parseDate()` 前先检查日期字符串格式
- 如果解析失败，使用 `now` 作为兜底

## 最佳实践

1. **错误兜底**：每个数据源的错误都应 try-catch，不影响其他数据源
2. **数据去重**：系统会自动去重，不需要在 Fetcher 中处理
3. **AI 过滤**：系统会自动进行 AI 相关性过滤，不需要在 Fetcher 中处理
4. **性能优化**：合理使用并发，避免给目标站点造成压力
5. **代码风格**：遵循现有代码风格，使用 TypeScript 类型注解

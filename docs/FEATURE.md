## 一、产品概览

**AI快讯** 是一款面向 AI 从业者和技术爱好者的**多源聚合式新闻阅读器**。它通过自动化技术聚合全球 13+ 主流科技平台、160+ 信息源的 AI 相关资讯，经过智能筛选、去重、翻译后，以时间轴形式呈现给用户，让你每天 5 分钟掌握 AI 行业最新动态。

## 三、核心功能详解

### 3.1 多源新闻聚合

#### 功能描述

聚合全球主流 AI 资讯平台，一站式获取所有动态。

#### 覆盖平台

| 平台类别 | 代表平台 | 说明 |
|:---------|:---------|:-----|
| 科技新闻 | Hacker News, TechCrunch, The Verge | 全球主流科技媒体 |
| AI 垂直 | AI News, MIT Technology Review | AI 领域专业媒体 |
| 产品发现 | Product Hunt | 新产品发布 |
| 社区讨论 | Reddit r/MachineLearning, 知乎 | 社区讨论热点 |
| 博客平台 | Medium, Towards Data Science | 深度技术文章 |
| 中文源 | 机器之心, 量子位, InfoQ 中文 | 中文 AI 媒体 |

#### 技术实现

- **多协议支持**：RSS / Atom / JSON Feed / HTML 抓取
- **智能去重**：标题相似度 + URL 规范化双重去重
- **AI 过滤**：基于关键词的 AI 相关性筛选
- **增量更新**：基于历史归档，只处理新数据

#### 使用场景

- 早晨通勤：快速浏览昨天 AI 圈发生了什么
- 工作间隙：看看有没有新的重磅发布
- 周末充电：用 7 天模式回顾一周热点

### 3.2 双语翻译系统 🌟 v2.0 新增

#### 功能描述

一键切换新闻标题的显示语言，支持中文、英文、双语三种模式，降低英文资讯阅读门槛。

#### 三种模式

| 模式 | 图标 | 显示效果 | 适用场景 |
|:-----|:----:|:---------|:---------|
| 中文模式 | 🇨🇳 | 仅显示中文标题 | 快速浏览，不想看英文 |
| 双语模式 | 🌐 | 中文 / 英文 对照显示 | 学习英文，对照阅读 |
| 英文模式 | 🇺🇸 | 仅显示英文标题 | 英文阅读能力强，追求原文 |

#### 交互方式

- **单按钮循环切换**：点击翻译按钮，按 中 → 双 → 英 → 中 循环
- **即时响应**：切换后所有新闻标题立即更新，无刷新
- **状态记忆**：自动记住用户选择，下次打开保持上次模式

#### 技术亮点

1. **后端预翻译**：数据抓取时即完成翻译，前端零延迟切换
2. **三级缓存**：
   - 一级：同 URL 中文标题（直接复用）
   - 二级：历史翻译缓存（持久化存储）
   - 三级：Google 翻译 API（新标题才调用）
3. **优雅降级**：翻译失败时自动显示原文，不影响使用

#### 使用场景

- 英文不好的用户：用中文模式无障碍阅读
- 想提升英文的用户：用双语模式边读边学
- 专业用户：用英文模式看原汁原味的标题

### 3.3 收藏功能 🌟 v2.0 新增

#### 功能描述

对感兴趣的新闻点击星标收藏，方便后续回看和整理。

#### 核心能力

| 能力 | 说明 |
|:-----|:-----|
| 一键收藏 | 新闻卡片右上角星标，点击即收藏/取消 |
| 收藏列表 | 弹窗查看所有收藏的新闻 |
| 新增计数 | 按钮上显示新增收藏数量，红点提醒 |
| 查看清零 | 打开收藏列表后，新增计数自动清零 |
| 单条删除 | 收藏列表中可删除单条 |
| 一键清空 | 快速清空所有收藏（需确认） |
| 数据导出 | 收藏数据可导出为 JSON 文件 |
| 本地持久化 | 刷新页面收藏不丢失 |

#### 新增计数机制

```
新增收藏数 = 当前收藏总数 - 上次查看时的收藏数

当你打开收藏弹窗：
  → 记录当前收藏数为"上次查看数"
  → 新增数变为 0
  → 下次有新收藏时，重新计算差值
```

**设计理念**：类似微信/钉钉的未读消息，让你不会错过任何一条新收藏的好文章。

#### 使用场景

- 看到感兴趣但没时间细读的文章，先收藏
- 收藏优质文章，建立自己的 AI 知识库
- 导出收藏数据，备份到其他地方

### 3.4 阅读历史 🌟 v2.0 新增

#### 功能描述

自动记录你点击过的新闻，方便回溯之前看到的文章。

#### 核心能力

| 能力 | 说明 |
|:-----|:-----|
| 自动记录 | 点击新闻链接自动加入历史 |
| 已读标记 | 已读新闻在列表中显示"已读"标签 |
| 历史列表 | 弹窗查看所有阅读历史 |
| 新增计数 | 按钮上显示新增已读数量 |
| 查看清零 | 打开历史列表后，新增计数自动清零 |
| 容量限制 | 最多保存 100 条历史记录 |
| 去重逻辑 | 同一篇文章重复点击只更新时间 |

#### 设计说明

- **最多 100 条**：超过自动删除最旧的记录，避免占用过多存储空间
- **自动去重**：同一篇文章多次点击，只保留最后一次阅读时间
- **隐私保护**：历史记录仅保存在本地浏览器，不上传任何服务器

#### 使用场景

- "刚才那篇文章叫什么来着？" → 翻历史记录
- 标记已读，避免重复点击
- 统计自己的阅读量（隐含）

### 3.5 滚动进度条增强 🌟 v2.0 新增

#### 功能描述

右侧竖向滚动进度条，支持点击和拖拽快速定位，长文阅读更高效。

#### 交互能力

| 交互方式 | 说明 |
|:---------|:-----|
| 点击跳转 | 点击进度条任意位置，页面平滑滚动到对应位置 |
| 拖拽跳转 | 按住进度条上下拖拽，页面实时跟随滚动 |
| 百分比显示 | 进度条中间显示当前滚动百分比 |
| 顶部按钮 | 点击 ↑ 一键回到顶部 |
| 底部按钮 | 点击 ↓ 一键跳到底部 |
| 智能显隐 | 页面顶部时隐藏，滚动后显示 |

#### 技术细节

- **精准定位**：基于 `getBoundingClientRect()` 计算坐标，误差 < 1%
- **流畅拖拽**：拖拽时禁用过渡动画，跟手性更好
- **平滑滚动**：点击跳转使用 `behavior: 'smooth'`，视觉舒适
- **性能优化**：滚动事件使用 `passive: true`，不阻塞渲染

#### 使用场景

- 快速定位到某天的新闻 → 点击进度条对应位置
- 浏览中途想回顶部 → 点击 ↑ 按钮
- 想快速滑到底部看更早的 → 点击 ↓ 按钮
- 精细调整位置 → 拖拽进度条

### 3.6 日期校验系统 🌟 v2.0 修复

#### 问题背景

部分数据源（如 NewsNow）的发布时间可能解析为未来日期，导致：
- 时间轴上出现"明天"的新闻
- 新发布的新闻排序异常
- 用户对数据准确性产生质疑

#### 修复方案

**1 小时宽容度原则**：

```
如果 发布时间 - 当前时间 > 1 小时：
  → 判定为无效日期
  → 使用抓取时间作为发布时间
  
否则：
  → 判定为正常（时区差异/时钟偏差）
  → 保留原始发布时间
```

**双层校验机制**：
1. **新数据校验**：每次抓取新数据时即时校验
2. **历史数据修复**：每次运行时批量修复归档中的异常数据

#### 效果

- 时间轴日期 100% 准确，不会出现未来的新闻
- 排序正确，最新的新闻始终在最前面
- 数据可信度提升

## 五、常见问题 FAQ

### Q1：翻译功能不生效怎么办？

**A**：翻译功能需要部署在可以访问 Google 的服务器环境。如果是本地开发环境，可能无法调用 Google 翻译 API。部署到外网环境后会自动生成翻译数据。

### Q2：收藏的数据会丢失吗？

**A**：收藏数据保存在浏览器的 localStorage 中，只要不清除浏览器数据就不会丢失。建议定期使用"导出"功能备份。

### Q3：新闻多久更新一次？

**A**：默认每 2 小时自动抓取一次。你也可以通过 GitHub Actions 手动触发更新。

### Q4：为什么时间轴上的日期和实际发布日期不一样？

**A**：v2.0 已修复此问题。如果发现日期异常，可能是数据缓存问题，刷新页面即可获取最新数据。

### Q5：支持自定义信息源吗？

**A**：目前通过 OPML 文件管理信息源，可以修改 `feeds/follow.opml` 文件添加自定义源，然后重新运行抓取脚本。

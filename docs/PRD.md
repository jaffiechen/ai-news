## 一、项目背景与目标

### 1.1 业务背景
AI 从业者和爱好者每天需要追踪全球 AI 领域的最新动态，但信息分散在多个平台（Buzzing、NewsNow、Hacker News、arXiv、微信公众号、RSS 订阅源等），来回切换耗费大量精力。用户需要一个聚合平台，用极简的方式获取干净、及时的 AI 资讯。

### 1.2 项目定位
**AI 快讯**是一个**全自动化的 AI 资讯聚合系统**，采用「无服务器架构 + 静态站点生成」模式，通过 GitHub Actions 定时从 13+ 聚合平台和 160+ 信息源抓取数据，经过智能过滤、去重、翻译后，以极简时间轴形式呈现给用户。

### 1.3 项目目标
打造一个沉浸式的 AI 新闻聚合时间轴，让用户只需一个页面就能按时间顺序追踪全球 AI 圈正在发生的大事，体验媲美刷朋友圈的沉浸感。

### 1.4 成功指标（Success Metrics）
| 指标类别 | 具体指标 | 目标值 |
|:--------|:--------|:------|
| **性能指标** | 首屏加载时间 | ≤ 2 秒 |
| | 时间轴滚动帧率 | ≥ 60 FPS |
| | 数据更新延迟 | ≤ 2 小时 |
| **功能指标** | 信息源数量 | ≥ 160 个 |
| | 聚合平台数量 | ≥ 13 个 |
| | 双语翻译覆盖率 | ≥ 80% 英文标题 |
| **用户体验** | PWA 可安装性 | 支持 iOS / Android / Desktop |
| | 暗色模式 | 支持自动 + 手动切换 |
| | 离线可用性 | 已缓存内容可离线访问 |

## 四、非功能需求

### 4.1 性能要求
| 指标 | 要求 |
|:-----|:-----|
| 首屏加载时间 | ≤ 2 秒（网络良好条件下） |
| 时间轴滚动帧率 | ≥ 60 FPS |
| 数据更新频率 | 每 2 小时自动更新一次 |
| 翻译 API 超时 | ≤ 12 秒（单条标题） |

### 4.2 安全与隐私
| 类别 | 要求 |
|:-----|:-----|
| 用户数据 | 不收集用户个人信息，所有数据仅存储在用户浏览器本地 |
| 数据传输 | 静态资源通过 HTTPS 传输 |
| 第三方资源 | 不下载第三方图片到服务器，直接引用原始链接 |
| 翻译服务 | 使用 Google 翻译公共 API，不存储用户翻译内容 |

### 4.3 兼容性
| 类别 | 要求 |
|:-----|:-----|
| 浏览器 | Chrome ≥ 90、Safari ≥ 14、Edge ≥ 90、Firefox ≥ 88 |
| 移动端 | iOS Safari ≥ 14、Android Chrome ≥ 90 |
| PWA | 支持添加到主屏幕，独立应用窗口 |
| 离线访问 | Service Worker 缓存已访问资源，离线可查看 |

### 4.4 可维护性
| 类别 | 要求 |
|:-----|:-----|
| 模块化 | 代码高度模块化，新增信息源无需修改核心逻辑 |
| 配置化 | 信息源配置支持 OPML 文件导入导出 |
| CI/CD | GitHub Actions 全自动数据更新和部署 |

## 五、数据需求

### 5.1 数据源架构
```
┌─────────────────────────────────────────────────────────────┐
│                      数据源层（13+ 平台）                     │
├─────────────┬───────────────┬───────────────┬───────────────┤
│  Buzzing    │   NewsNow     │   TopHub      │   Info Flow   │
│  (110条)    │   (35条)      │  (141条)      │   (191条)     │
├─────────────┼───────────────┼───────────────┼───────────────┤
│   Zeli      │  TechURLs     │   OPML RSS    │  微信公众号   │
│  (66条)     │  (163条)      │   (63条)      │   (56条)      │
├─────────────┼───────────────┼───────────────┼───────────────┤
│   AIbase    │  AI HubToday  │   新智元      │   YouTube     │
│  (38条)     │   (11条)      │   (15条)      │    (1条)      │
└─────────────┴───────────────┴───────────────┴───────────────┘
```

### 5.2 数据处理流程
1. **采集**：从各平台 API / RSS 源抓取原始数据
2. **过滤**：AI 相关性过滤，仅保留 AI / 科技相关内容
3. **去重**：基于标题和 URL 去重，避免重复内容
4. **翻译**：英文标题自动翻译为中文，生成双语字段
5. **输出**：生成静态 JSON 文件，供前端加载

### 5.3 数据字段规范
每条新闻包含以下核心字段：
| 字段 | 类型 | 说明 |
|:-----|:-----|:-----|
| `id` | string | 唯一标识符（SHA-1 哈希） |
| `title` | string | 原始标题 |
| `title_zh` | string | 中文标题（翻译或原始中文） |
| `title_en` | string | 英文标题（原始英文或 null） |
| `title_bilingual` | string | 双语标题格式：「中文 / 英文」 |
| `url` | string | 原文链接 |
| `published_at` | string | 发布时间（ISO 8601 格式） |
| `first_seen_at` | string | 首次抓取时间 |
| `last_seen_at` | string | 最后出现时间 |
| `site_id` | string | 聚合平台 ID |
| `site_name` | string | 聚合平台名称 |
| `source` | string | 具体来源名称 |
